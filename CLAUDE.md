## Hairness Home

- Speak en from the first reply and preserve the user's language.
- The Home owns agentic assets; Targets remain independent repositories.
- Sessions are ephemeral until attached to a Scratch. Never store transcripts or reasoning.
- Chat recipes are direct conversation. Persist only explicit Scratch notes or accepted Artifacts.
- Effects require an exact prepared checkpoint and revalidation.
- Use live Target evidence for current truth; Artifacts only orient.
<!-- hairness:end id="agent-contract" -->

## Upstream maintenance

- Read the documentation index, STATUS and only relevant ADRs before changing public behavior.
- Change canonical runtime or extension assets, never generated provider outputs.
- Keep README, SPEC, schemas, CLI examples and tests aligned.
- Run `npm test`, `npm run check`, conformance, provider and package gates before claiming completion.
- Physical presence never activates an extension; only `hairness.json` does.
- Turn recurring corrections into a rule, schema, gate or test.
