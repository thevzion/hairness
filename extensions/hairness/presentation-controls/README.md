# hairness/presentation-controls

## Value and use cases

Chooses the smallest useful views for understanding or resuming a result.

## Selection and setup

Selected by standard and forge and used only by operations that opt into its modifier.

## Capabilities and operations

Owns `present` policies and the summary, diagram, tree, table, timeline, checklist, matrix and trace views.

## Inputs, controls and results

Accepts `auto`, `compact`, `visual`, `explicit` or an explicit view.

## State and artifacts

Presentation changes no canonical result and owns no semantic artifact.

## Effects and safety

It may change form but cannot invent meaning, structure, decisions or proof.

## Providers

Compiled as an accepted modifier rather than a standalone provider command.

## Tests and maturity

Official alpha. Tests cap automatic presentation at three views.
