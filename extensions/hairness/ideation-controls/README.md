# hairness/ideation-controls

## Value and use cases

Gives the main session explicit divergent, lateral and convergent creative strategies.

## Selection and setup

Selected by standard and forge and composed with Presentation Controls.

## Capabilities and operations

Owns ideate and propose operations plus the `creative` modifier.

## Inputs, controls and results

Accepts a topic and creative strategy and returns a typed semantic result.

## State and artifacts

No state is persisted unless the selected result declares an artifact.

## Effects and safety

Ideation cannot invent proof or grant effects.

## Providers

Projects `hairness-x-propose` as the provider-facing recommendation intent. `ideate` remains available as an internal route.

## Tests and maturity

Official alpha. Tests keep creative strategy explicit and bounded.
