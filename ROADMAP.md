# Roadmap

Hairness develops from observed use, not speculative orchestration.

## v0.3 alpha

- qualify the clean architectural reset;
- publish `0.3.0-alpha.0` from a separate release PR;
- dogfood Standard against independent repositories;
- prove extension installation and divergence handling with third-party assets;
- harden adaptive checkout evidence and cleanup from real delivery flows.

## Later, only with proof

- additional provider compilers that preserve the same extension grammar;
- richer typed Artifact schemas owned by extensions;
- organization-specific distributions such as Hupso, outside the generic package;
- optional remote Overlay workflows only after local ownership and security are
  proven.

Hairness does not plan to become a provider runtime, scheduler, hosted registry,
shared transcript store, generic workflow engine, issue bot, worktree controller,
or package-manager abstraction.
