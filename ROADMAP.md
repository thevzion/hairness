# Roadmap

Hairness develops from concrete Homes, not speculative orchestration.

## v0.4 alpha

- prove the Kernel with one private Home and three independent Git Targets;
- prove one provider Integration shared by Codex and Claude;
- prove a two-file extension, a Command distinct from its Skill and one bounded
  prologue contribution;
- harden only failures observed in packed-tarball dogfood.

## Later, only with proof

- migrate the Hupso harness as a separate consumer;
- explore source-owned runtime installation;
- add typed Actions, Artifact Types or delivery gates only for a repeated need;
- add another provider compiler only if it preserves the same neutral assets.

Hairness does not plan to become a provider runtime, scheduler, hosted registry,
transcript store, generic workflow engine, worktree controller or package-manager
abstraction.
