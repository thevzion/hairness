# Hairness maintainer contract

- Preserve the user's language.
- Read the documentation index, STATUS and relevant decision before changing a
  public contract.
- Change canonical source, schemas and package assets. Do not edit generated
  provider output as source.
- Keep README, SPEC, schemas, CLI and tests aligned.
- Physical package presence does not activate an Extension.
- Targets remain independent Git repositories.
- Persist no transcript, hidden reasoning, credential or private downstream
  information.
- Run tests, check, conformance, provider, package and lab gates before claiming
  completion.
