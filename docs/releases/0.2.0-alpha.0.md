## What this alpha proves

- Agentic assets have extension-owned capabilities, typed operations, routes
  and results.
- One deterministic Invocation Engine serves natural provider commands and
  direct CLI automation.
- A provider-independent CommandSurfaceSpec compiles 29 standard commands and
  five forge delivery commands with Codex/Claude parity.
- The Semantic Ledger, shared Run roots, fan-in and Attention Index preserve
  operational continuity without transcripts.
- Typed make-to-save promotion preserves exact SegmentDigest and WorkPlan
  payloads without resynthesis.
- Versioned migration planning transforms owned local state in scratch and
  records accepted migration digests in the distribution lock.
- Codex and Claude receive the same selected operations through repo-local
  projections.
- Minimal, standard and forge distributions contain only their resolved
  material closure.
- Effects remain behind exact authority, current policy, target and result
  gates.
- A forge can plan initiatives and delivery without granting Git, package or
  publication authority to its handlers.
- Policy-driven GitHub Flow turns accepted briefs into parallel change plans,
  then aggregates merged changes into a separately checkpointed release plan.
- Stored authority checkpoints, credential-free URI locks, read-only GitHub/npm
  proof and correlated PR/release artifacts make each external boundary
  inspectable and recoverable.
- Release plans require exact-head pull-request, CI, squash-merge, public-main
  and tarball qualification receipts before npm publication can be proposed.
- Partial, failed or unknown receipts remain immutable and require a separate,
  exact reconciliation checkpoint before delivery can continue.

## Evaluation path

1. Install the exact release tarball under Node.js 22 or 24.
2. Create a temporary standard distribution.
3. Complete onboarding and open a new trusted provider task.
4. Exercise wake-up, topics, resume, discussion, creative intent and one
   bounded producer.
5. Promote one artifact, inspect its trace, reconcile an interruption, and
   migrate a fixture from the previous release state.
6. Inspect invocation, artifact, fan-in, migration and provider doctor receipts.

## Not included

- A stable `latest` channel or pre-1.0 compatibility guarantees
- Remote extension registry or package manager
- Generic execution-loop adapter
- Telemetry or transcript storage
- Compatibility with unpublished protocol `0.1`
