# Intent composition

Hairness keeps primitives narrow, then composes them under one human command.

```text
operation + focus + source policy + boundary + execution mode + budget + modifiers
```

For example:

```text
map
+ focus: structure
+ sources: prove
+ boundary: read-only
+ present: auto
```

The operation owns the meaning. Sources own proof. Constraints own the boundary. Presentation owns the rendering form. A provider quick command gives that composition a memorable human name; it does not create another capability.

The main session resolves semantic choices because model inference adds value there. The CLI validates known arguments, active owners, budgets, constraints, checkpoints, and result contracts.

This is the leverage behind “Move mountains with one intent”: the human does not micromanage routes, and the agent does not improvise authority.
