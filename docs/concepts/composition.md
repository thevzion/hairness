# Intent composition

Hairness keeps operations narrow and composes them under one human intent:

```text
operation + focus + source policy + boundary + route + budget + modifiers
```

The main-session model resolves semantic choices where inference helps. Deterministic gates validate active owners, operation class, supported route, source selection, effect policy, target, result, and fan-in.

This is “move mountains with one intent” without a magic prompt: the human does not micromanage transport, and the agent cannot improvise authority or proof.

Cross-extension behavior follows declared service dependencies. Managed
delivery composes in one direction:

```text
delivery-controls → worktree-controls → codebase + sources + session-intelligence
```

Delivery decides when a checkout is needed; Worktree owns its lifecycle;
Sources proves Git state; Codebase exposes authority-asserted mounts; Session
Intelligence supplies session identity. No reverse dependency or kernel GitHub
Flow special case is introduced.
