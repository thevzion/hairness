# Composable controls

Controls are small cognitive and operational capabilities that combine under one intent.

```text
operation
+ focus
+ source policy
+ boundary
+ execution mode
+ budget
+ accepted modifiers
```

The standard controls are:

- Work: discuss, recap, plan, act, execute and trajectory;
- Understanding: map, explain, compare;
- Ideation: ideate, propose, divergent/lateral/convergent strategy;
- Presentation: auto, compact, visual, explicit and eight views;
- Constraints: inherited restrictions on effects.

For example, “map this subsystem visually using current proof” can resolve to `map + focus: subsystem + sources: prove + boundary: read-only + present: visual`. It remains one human intent and one operation owner.

Modifiers change strategy or form. They cannot add evidence, authority, or decisions that the underlying operation did not produce.

Work Controls may persist ordinary controls at session, segment or frame scope. Effective controls resolve from broad to narrow, and an operation override wins last. Constraints remain a separate monotonic boundary: a child may tighten them but never loosen them. `hairness work control show` explains both the local and effective values without asking the agent to read overlay files.
