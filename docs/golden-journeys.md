# Golden journeys

## Personal Home

1. Pack the single CLI candidate.
2. Create a Home without `package.json`, `package-lock.json` or `node_modules`.
3. Confirm `hairness/core` is tracked as source with a provenance receipt.
4. Run build check, doctor and prologue.
5. Confirm Codex and Claude projections are equivalent.

## Source-owned Extension

1. Preview and add a local, HTTPS, namespace or GitHub item.
2. Confirm installation executes no code.
3. Edit a declared file and observe `customized` offline.
4. Confirm sync blocks without partial writes and `--overwrite` is explicit.
5. Confirm unknown files survive sync and remove.
6. Confirm dependencies, cycles, paths, symlinks and collisions are enforced.

## Team Home with GSD

1. Add the GSD source registry item without execution.
2. Confirm ordinary build refuses it.
3. Build with `--allow-adapter hairness-gsd`.
4. Confirm the official installer produced GSD Core `1.6.1` in staging.
5. Confirm build check and doctor are ready afterward.

## Existing Homes

1. Migrate each Home on its own branch and worktree.
2. Replace only the old Hairness package graph and Home document.
3. Preserve all Targets, Integrations and Overlay files.
4. Add the source-owned core, bind existing Targets and build.
5. Confirm doctor is ready and no Hairness package dependency remains.
